# Skill: handoff

Use this when passing work to another agent.

1. Identify the receiving agent (api-dev, frontend-dev, architect, qa-tester, or project-manager).
2. Write a handoff note in `team/<receiving-agent>/memory.md`:
   - What was completed
   - What the receiving agent needs to do next
   - Any decisions or constraints they must know
   - The task ID and RD reference
3. Update the task status in `data.json` if appropriate.
4. Keep the note under 10 lines — clarity beats completeness.
